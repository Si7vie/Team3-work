## 二、这些概念是怎么串联起来的？

整个项目就是一条**流水线**，像工厂的传送带一样：

```
📄 PDF公告（网上的原始文件）
        ↓  ① 通过 MinerU API 提交
🤖 MinerU 云端解析（AI把PDF转成文字）
        ↓  ② 下载解析结果
📝 Markdown 文件（纯文字版公告）
        ↓  ③ build_parsed_docs.py 组装
📦 parsed_docs.jsonl（把所有MD文件整理成一个大文件）
        ↓  ④ route_sections.py 拼接
📋 sections.jsonl（每篇公告的全文合并到一起）
        ↓  ⑤ extract_fields.py 提取（正则匹配）
🔢 extract_results.jsonl（7个字段的原始提取结果）
        ↓  ⑥ validate_results.py 校验
✅ records_validated.csv（最终的结构化表格）
```

> **一句话总结**：PDF → MinerU转文字 → 拼成全文 → 用正则提取数字 → 用Pydantic校验 → 得到干净的CSV表格。

## 三、第一步：PDF → Markdown（MinerU 是怎么工作的）

### 3.1 什么是 MinerU？

MinerU 是**开放解析实验室（OpenXLab）** 提供的一个AI文档解析服务。你可以把它理解成一个"PDF阅读机器人"：

- **输入**：一个 PDF 文件的网址
- **输出**：一个 Markdown 文件（纯文字 + 图片链接）

### 3.2 为什么需要它？

PDF 文件里的文字不是直接能拿来用的——有些是扫描件、有些排版复杂、有些表格嵌套很深。MinerU 能：

1. 识别 PDF 里的所有文字
2. 保留标题、段落、表格的结构
3. 提取里面的图片并保存

### 3.3 代码是如何调用 MinerU 的？

## 五、第三步：Schema 定义（数据模板）

### 5.1 什么是 Schema？

Schema 就是**数据模板**。它规定了：
- 每个字段**叫什么名字**
- 每个字段是**什么类型**（数字？文字？）
- 哪些字段**必须有**，哪些**可以没有**

### 5.2 这个项目的 Schema

```python
# 📄 文件：src/schemas.py

from pydantic import BaseModel, Field
from typing import Optional

# 证据类：保存提取字段时引用的原文
class Evidence(BaseModel):
    text: str                    # 原文片段
    page_no: Optional[int] = None  # 页码（可选）

# 核心数据模型：一份股权激励公告的结构化信息
class EquityIncentiveExtract(BaseModel):

    doc_id: str                  # 公告ID（必须有）

    company_name: str            # 公司名称（必须有）

    stock_code: Optional[str] = None  # 股票代码（可有可无）

    # 📊 字段1：拟授予总数量
    grant_amount: float = Field(description="拟授予总数量（万份）")

    # 📊 字段2：占总股本比例
    grant_ratio: Optional[float] = Field(description="占总股本比例（%）")

    # 👥 字段3：激励总人数
    participant_count: int = Field(description="首次授予总人数")

    # 💰 字段4：行权价格
    exercise_price: float = Field(description="行权价格（元/份）")

    # 💰 字段5：交易均价（用于计算折扣率）
    market_price: Optional[float] = Field(description="公告日前1个交易日股票均价（元）")

    # 📉 字段5衍生：折扣率
    discount_rate: Optional[float] = Field(description="定价折扣率（%）")

    # ⏰ 字段6：等待期
    waiting_period: Optional[float] = Field(description="等待期（月）")

    # ⏰ 字段7：有效期
    validity_period: Optional[float] = Field(description="激励计划最长有效期（月）")

    # 📎 提取证据（保留原文，方便人工核对）
    evidences: list[Evidence]
```

### 5.3 为什么需要 Schema？

Schema 就像一个**安检门**：
- 如果提取出来的 `grant_amount` 是 "三千份" 这样的文字，就会被拦下（因为模板要求是数字）
- 如果某项数据缺失，模板会标记为 None，不会导致程序崩溃
- **自动校验**：validate_results.py 会利用这个模板检查每一条数据是否合格

## 七、第五步：字段提取（extract_fields.py）—— 整个项目的核心！

这是**最关键的一步**：从几万字的公告全文里，精准地找出 7 个数字字段。

### 7.1 提取方式：正则表达式（Regex）

**什么是正则表达式？** 它是一种"文字搜索模式"。类比：你在一本书里用 Ctrl+F 搜索关键词，但正则比 Ctrl+F 更聪明——它能搜索"模式"而不仅仅是"固定的字"。

比如：`([0-9,.，]+)\s*万份` 这个表达式能匹配到 "3,186.2261 万份"、"1200 万份"、"50万份" 等各种写法。

### 7.2 总调度函数

### 7.3 逐字段讲解提取逻辑

#### 字段1：grant_amount（拟授予总数量，单位：万份）

```python
def extract_grant_amount(text: str) -> float | None:
    patterns = [
        # 模式1："授予...共计 3,186.2261 万份"
        r"授予(?:的)?(?:股票期权|股票|权益).*?(?:共计|合计|共)\s*([0-9,.，]+)\s*万[份股]",

        # 模式2："拟向激励对象授予...1200万份"
        r"拟向激励对象授予.*?([0-9,.，]+)\s*万份",

        # 模式3："授予股票期权总量 2000 万份"
        r"授予(?:的)?股票期权(?:总量|数量).*?([0-9,.，]+)\s*万份",

        # ... 共15条规则 ...

        # ⚠️ 关键规则：不带"万"字的，单位是"份"，必须除以10000！
        r"授予(?:的)?股票期权总量.*?([0-9,.，]+)\s*[份股]",
    ]

    for p in patterns:
        m = re.search(p, text, re.S)  # 在全文里搜索
        if m:
            value = float(m.group(1).replace(",", "").replace("，", ""))
            matched = m.group(0)

            # 🔑 关键判断：如果原文没写"万份/万股"，说明是"份/股"
            if "万份" not in matched and "万股" not in matched:
                value = value / 10000  # ← 除以10000转为万份

            return round(value, 4)

    return None  # 找不到就返回空
```

**容易踩的坑**：同一份公告里，"万份"和"份"两种写法都有。比如有的写"3,186.2261 万份"，有的写"5,726,100 份"。不判断就提取的话，5,726,100（万份）明显错的——应该是572.61万份。

#### 字段2：grant_ratio（占公告日总股本比例，单位：%）

```python
def extract_grant_ratio(text: str) -> float | None:
    patterns = [
        r"约占.*?总股本.*?([0-9.]+)\s*%",   # "约占...总股本...3.5%"
        r"占.*?股本总额.*?([0-9.]+)\s*%",    # "占...股本总额...2.8%"
        r"占.*?股份总数.*?([0-9.]+)\s*%",    # "占...股份总数...1.5%"
        # ... 共10条规则 ...
    ]

    for p in patterns:
        # 🔑 关键：只在"授予"/"激励"关键词附近1000个字符内搜索
        grant_contexts = [m.start() for m in re.finditer(r"授[予与]|激励", text)]

        for m in re.finditer(p, text, re.S):
            val = float(m.group(1))
            pos = m.start()

            # 检查是否在授予相关上下文附近
            near_grant = any(abs(pos - gc) < 1000 for gc in grant_contexts)
            if near_grant and val <= 50:  # 合理范围：50%以下
                return val

    return None
```

**为什么要在"授予附近"搜索？** 因为一篇公告里会提到很多百分比（比如持股比例、业绩增长率等），只有"授予"附近的百分比才是激励计划的比例。

#### 字段3：participant_count（首次授予总人数）

```python
def extract_participant_count(text: str) -> int | None:
    patterns = [
        r"首次授予(?:的)?激励对象(?:共计|共|总人数为)\s*([0-9,，]+)\s*[人名]",
        # ↑ "首次授予的激励对象共计 123 人"

        r"激励对象总人数(?:为)?\s*([0-9,，]+)\s*人",
        # ↑ "激励对象总人数为 456 人"

        r"激励对象共计\s*([0-9,，]+)\s*[人名]",
        r"共\s*([0-9,，]+)\s*名激励对象",
        r"合[计計]\s*([0-9,，]+)\s*[人名]",
        # ↑ 表格里的"合计 6302 人"

        # ... 共15条规则 ...
    ]

    for p in patterns:
        m = re.search(p, text, re.S)
        if m:
            # ⚠️ 去除千分位逗号："6,302" → 6302
            val = int(m.group(1).replace(",", "").replace("，", ""))
            if 0 < val < 100000:  # 合理性检查：1~10万之间
                return val

    return None
```

**容易踩的坑**：部分公告人数用了千分位逗号（如"6,302 人"），正则如果只写 `[0-9]+` 就只能抓到逗号前的"6"，漏掉"302"。

#### 字段5：market_price（公告前1日股票交易均价）—— 最难的一个！

```python
def extract_market_price(text: str) -> float | None:
    """
    三层优先级匹配策略（从 report 第四章总结的坑）:
      优先级1: "每股X元 的 Y%" → X 就是市场价
      优先级2: "交易均价(的) Y%，为/即 Z元" → Z ÷ (Y/100) 反推
      优先级3: 直接价格声明 → 直接提取
    """

    # 第一步：找到所有"前1个交易日"+"交易均价"同时出现的上下文
    segments = []
    anchor_pattern = re.compile(r'前\s*(?:1|一)\s*个?\s*交\s*易\s*日')

    for anchor in anchor_pattern.finditer(text):
        # 提取从"前1个交易日"开始的约600个字符
        window = text[anchor.start():anchor.start() + 600]

        # ⚠️ 如果包含"前20/60/120个交易日"，跳过！（不是我们想要的1日）
        if re.search(r'前\s*(?:20|60|120)\s*个\s*交\s*易\s*日', window):
            continue

        if '交易均价' in window:
            segments.append(window)

    # 第二步：在每个上下文片段中按优先级提取
    for segment in segments:

        # === 优先级1: "每股90.49元的50%" → market_price = 90.49 ===
        m1 = re.search(
            r'(?:每股\s*([0-9.]+)\s*元|([0-9.]+)\s*元\s*/\s*股)'
            r'[^。；\n]{0,60}?的\s*([0-9.]+)\s*%',
            segment
        )
        if m1:
            return float(m1.group(1) or m1.group(2))

        # === 优先级2: "交易均价的75%，为每股7.47元" → 7.47÷0.75=9.96 ===
        m2 = re.search(
            r'交易均价[^。；\n]{0,60}?的\s*([0-9.]+)\s*%'
            r'[^。；\n]{0,80}?'
            r'(?:为每股|即每股|为|即)\s*([0-9.]+)\s*元',
            segment
        )
        if m2:
            pct = float(m2.group(1)) / 100   # 75% → 0.75
            discounted_price = float(m2.group(2))
            if pct > 0:
                return round(discounted_price / pct, 2)  # 反推市场价

        # === 优先级3: 直接匹配"为每股X元"等 ===
        m3 = re.search(r'为每股\s*([0-9.]+)\s*元', segment)
        if m3:
            return float(m3.group(1))

    return None
```

**这是整个项目最复杂的提取**，因为不同公司的公告对"交易均价"的表述差异很大：

| 情况 | 原文示例 | 市场价 |
|------|----------|--------|
| A-直接给出 | "交易均价**为每股 9.03 元**" | 9.03 |
| B-打折后再给 | "交易均价**的 75%**，**为每股 7.47 元**" | 7.47÷0.75=**9.96** |
| C-先给原价再说折后价 | "交易均价**每股 90.49 元**的 **50%**，为每股 45.25 元" | **90.49** |

如果搞错情况B，就会把折后价7.47当成市场价，导致折扣率算错。

#### 字段5衍生：discount_rate（折扣率）—— 不是提取，是计算

```python
def calculate_discount_rate(
    exercise_price: float | None,
    market_price: float | None,
) -> float | None:
    if exercise_price is None or market_price is None or market_price <= 0:
        return None

    # 公式：(1 - 行权价格 / 市场价) × 100
    discount_rate = round((1 - exercise_price / market_price) * 100, 2)

    # 过滤异常值
    if discount_rate < -100 or discount_rate > 100:
        return None
    return discount_rate
```

**例如**：行权价=9.03，市场价=10.00 → 折扣率 = (1-9.03/10) × 100 = **9.7%**（员工能以9.03元买到市场价10元的股票，相当于打了9折）

#### 字段6：waiting_period（等待期，单位：月）+ 字段7：validity_period（有效期）

```python
def extract_waiting_period(text: str) -> float | None:
    patterns = [
        # 多阶段："等待期分别为 12个月、24个月、36个月" → 取12
        r"等待期分别[为是]?\s*[^。；\n]{0,100}?([0-9]+)\s*个?月[、，和及]",

        # 单阶段："等待期为 12 个月"
        r"等待期[为是]?\s*([0-9]+)\s*个月",

        # "自授予登记完成之日起12个月"
        r"自[^。；\n]{0,60}?(?:授予|登记|授权)[^。；\n]{0,30}?(?:之日|日)?起[^。；\n]{0,60}?([0-9]+)\s*个?月",
    ]

    for p in patterns:
        m = re.search(p, text, re.S)
        if m:
            val = float(m.group(1))
            if 1 <= val <= 120:  # 合理范围：1~120个月
                return val
    return None


def extract_validity_period(text: str) -> float | None:
    patterns = [
        r"有效期.*?最长.*?([0-9]+)\s*个月",
        r"有效期最长.*?([0-9]+)\s*个月",
        r"有效期.*?不超过\s*([0-9]+)\s*个月",
        r"有效期[为是]?\s*([0-9]+)\s*个月",
        r"最长不超过\s*([0-9]+)\s*个月",
    ]

    for p in patterns:
        m = re.search(p, text, re.S)
        if m:
            return float(m.group(1))
    return None
```

    config = read_yaml(config_path)

    rows = read_jsonl("outputs/results/extract_results.jsonl")
